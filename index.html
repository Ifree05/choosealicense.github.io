开源许可用于保护这些开源项目的使用者和使用者。如果没有这些许可证的保护，商人和开发者都无法接触到这些项目。

{ 以下哪种情况最符合您的要求？ }

  • 社区协作开发

    使用在开源社区协作开发中首选的开源协议 无论你是在社区中作为一个贡献者或者是使用者，你都需要遵守许可协议。

    如果其中没有您所需要的许可证，请要求维护人员添加许可证 添加学科证.

  • 条款简单的许可证

    MIT License 非常简洁并且许可证较为宽松。 它基本允许开发者在开源项目上做任何想做的事情，包括分发闭源版本。

    {% include using-sentence.html license-id="mit" %}

  • 注重于项目的分享

    GNU GPLv3 同样基本允许开发者在开源项目上做任何想做的事情，但是禁止将其分发为闭源的版本。

    {% include using-sentence.html license-id="gpl-3.0" %}
